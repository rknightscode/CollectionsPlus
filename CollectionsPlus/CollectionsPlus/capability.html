Overview

A complete collections manager for Edge

Save pages, images, and snippets into named collections. Everything is stored locally in a single database file you control.

📁
Colour-coded collections
Organise saved pages into named collections, each with an optional description and colour.
📸
Automatic screenshots
Captures a screenshot of every page you save. Refresh it any time from the item row.
📝
Notes on every item
Add your own notes to any saved page, image, or snippet. Notes are searchable.
🔍
Search and filter
Search across collection names or within an open collection by title, URL, or notes.
↕
Drag and drop reorder
Reorder both collections and items within a collection by dragging or using arrow buttons.
↗
Open all in one click
Open an entire collection as a named, colour-matched tab group with a single click.
☁
OneDrive backup
Back up and restore your entire database to OneDrive manually, on your terms.
🔒
100% private
No accounts, no servers, no telemetry. Everything stays in a single file on your computer.
Saving items

Multiple ways to save

Save to a collection however you prefer — from the sidebar, the right-click menu, or by importing in bulk.

  • Open the sidebar and click + Add Current Page to save the active tab with a screenshot
  • Right-click any page → Save page to Collections+ → choose a collection — works without opening the sidebar
  • Right-click any image → Save image to Collections+
  • Right-click any link → Save link to Collections+
  • Import directly from an Edge Collections SQLite export file
  • Paste copied Edge Collections content directly into the import tool
  • Use Suggestions to browse your recent browsing history and batch-add pages to any collection
Right-click saves: Screenshots are captured immediately at click time using the active tab. The item appears in the collection the next time you open the sidebar.
History suggestions

Add pages from your browsing history

The Suggestions view shows recent browsing history as quick-add candidates — useful for catching up on pages you forgot to save.

  • Filter by time range: Today, Last 3 Days, or all available history
  • See visit count and last visited date for each page
  • Select multiple pages with checkboxes, then add them all to a chosen collection at once
  • Duplicates are automatically skipped — pages already in the target collection are not added again
  • Non-HTTP pages (browser settings, search results) are filtered out automatically
Backup & restore

OneDrive backup

The entire database is a single file. Backup copies it to a OneDrive folder you choose; restore replaces your local copy.

  • Choose a OneDrive backup folder once in Settings — a backup is created immediately
  • Back up at any time with one click from the Settings panel
  • Restore shows the backup date and file size before you confirm
  • Use backup and restore to move your collections to a second computer
Note: Backup and restore are manual. Collections+ does not sync automatically — back up on one machine and restore on the other when you want to transfer changes.

Updates

Automatic update notifications

Collections+ checks for new releases on GitHub and shows a banner when one is available.

  • Checks are made at 09:00, 12:00, 18:00, and 21:00 daily using a background alarm
  • An update banner appears on the unlock screen when a newer version is available
  • The banner links directly to the GitHub releases page to download the new version
  • No automatic installation — you choose when to update
Privacy

Your data, your machine

  • All data is stored in a single collections.db SQLite file in a folder you choose
  • Nothing is sent to any external server except the GitHub version check (no data included — read-only API call)
  • The database can be opened and inspected independently with DB Browser for SQLite
  • Deleting the extension and its database folder removes everything completely
